    FAQ¶

    Quick answers to common questions. Each links to the fuller explanation.

    Getting started¶

    What is Vibrator, in one sentence?

    A single Go binary (vibrate) that builds a tailored Docker image per project and runs an AI coding agent — one of four harnesses (Claude Code, Codex, OpenCode, or Pi) — inside it, configured by a .vb file. See the overview.

    What's the difference between vibrate and vb?

    Nothing — vb is a symlink to vibrate, installed alongside it. Use either.

    Do I need Docker installed?

    To run a workspace, yes — any Docker-compatible runtime (Docker Desktop, OrbStack, Colima, Rancher Desktop, Podman, native). To merely build the binary, no. See Installation.

    How do I start?

    cd into a project and run vibrate. The wizard asks a few questions, the image builds, and the agent drops you in. See the Quick start.

    Daily use¶

    Why was the first run slow and the next one instant?

    The first run does a docker build for your variant. Once an image exists, vibrate reuses the container (or starts it) and just execs in. See Lifecycle.

    How do I change my setup after the first run?

    vibrate reconfigure re-runs the wizard and rebuilds, preserving your credentials and [env]. Or edit .vb by hand.

    How do I get a shell instead of the agent?

    vibrate shell. Same build/run logic, it just execs your shell at the end.

    How do I update the agent without rebuilding everything?

    vibrate update upgrades just the harness CLI in place. For a full rebuild from the Dockerfile, use vibrate --rebuild.

    How do I skip the wizard?

    Pass everything as flags and add --no-wizard: vibrate --no-wizard --harness=claude-code --profile=full. See the launch flags.

    How do I try a config once without saving it?

    Add --no-save — the resolved config isn't written to .vb.

    Configuration¶

    Where does Vibrator look for .vb?

    It walks up from your current directory to the git root (or filesystem root); the first .vb wins, and its directory becomes the workspace root. See The .vb file.

    Is .vb safe to commit?

    No — it can hold plaintext credentials (minted keys, pasted API keys). It's written 0600 and auto-added to .gitignore. Treat it like a dotenv file.

    What's a profile vs. a feature?

    A profile is a named bundle of features; features are the build-time layers (Python, Node, Playwright, …). Tune a profile with --with/--no. See Profiles & features.

    I removed a feature with --no but it's still there. Why?

    Something else needs it. --no=node --with=playwright keeps node because Playwright depends on it. Drop the dependent too. See the resolution subtlety.

    Why do --profile=full and the default produce the same image?

    Profile is just a label for a feature bundle; it's deliberately excluded from the fingerprint. Identical resolved features → identical image.

    Authentication¶

    The banner says auth: not configured. What now?

    Export the relevant key on the host before running (e.g. export ANTHROPIC_API_KEY=...), or for Claude Code run vibrate --login for the browser OAuth flow. See Authentication.

    What does vibrate --login do?

    Runs claude auth login in the container, opens the OAuth URL in your host browser, and writes the auth back to your host so future runs are pre-authenticated. See --login.

    Can I sign git commits inside the container?

    Yes — if you've configured a gpg-agent extra-socket, it's auto-forwarded so git commit -S uses your host key without it leaving the host. See GPG forwarding.

    Are my AWS credentials available?

    If ~/.aws exists on the host, it's mounted read-only. See AWS credentials.

    Models & extensions¶

    Can I use a local model (Ollama / LM Studio)?

    Yes, for Codex/OpenCode/Pi. Pick it in the wizard; Vibrator ensures the provider is running and pulls the model before launch. See LLM providers.

    How do I add an MCP server / skill / plugin?

    Select an extension by ID: --extensions=context7,github-mcp. Browse with vibrate extensions list <harness>.

    What is ECC?

    Everything Claude Code — an opt-in bundle of agents, skills, rules, and hooks, shipped as ecc-* extensions. Try --extensions=ecc-developer.

    Integrations¶

    What are integrations vs. extensions?

    Extensions are installed into the image. Integrations connect the container to a host-side service (Serena, claude-mem) with automatic transport fallback.

    What's the difference between auto, host, local, and off?

    They're hosting modes for an integration: probe-then-fall-back, host-only, container-only, or disabled.

    Troubleshooting¶

    vibrate can't find my Docker socket.

    Run vibrate runtime detect. Override with --docker-socket=... or VIBRATOR_DOCKER_SOCKET. See Runtime detection.

    I get node: not found (or python) errors from hooks.

    A hook in your ~/.claude/settings.json shells out to a tool that isn't installed in the image — common with the minimal profile. vibrate detects this at launch and offers to install the tool (adds --with=node, rebuilds) or disable those hooks, and the container also auto-skips any hook whose tool isn't on PATH. See Missing-tool hooks.

    How do I see what the container setup is doing?

    Set VIBRATOR_VERBOSE=1 to print [vibrator] ... diagnostics from the entrypoint and claude-exec. Silence the banner with VIBRATOR_NO_BANNER=1.

    How do I clean up old images and containers?

    vibrate variants list to see them, vibrate variants prune to remove stopped containers and unused images.

    The container can't reach a service on my host.

    Use host.docker.internal (Vibrator adds the host-gateway mapping automatically). For host integration servers, check they bind 0.0.0.0 and your hosting mode.

    Can the agent run docker itself?

    Yes, with --dind — but it grants host-root-equivalent access, so only use it for trusted workspaces.
