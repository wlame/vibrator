    Vibrator¶

    Run AI coding agents in disposable, per-workspace Docker sandboxes — configured by a single .vb file.

    Vibrator (vibrate, alias vb) is a single Go binary that builds a tailored Docker image for each of your projects and drops you straight into an AI coding agent — Claude Code, Codex, OpenCode, or Pi — running inside it. The agent gets a full toolchain, your host credentials, and your project mounted at the same absolute path, while staying isolated from the rest of your machine.

    cd ~/my-project
vibrate          # wizard fills the gaps, image builds, agent drops you in

    The first run writes a .vb file capturing your choices, so every subsequent vibrate in that workspace reuses the container and jumps in instantly.

    Why use it?¶

    • Isolation per workspace

      Each project gets its own image and container, keyed by a content fingerprint. Two projects never share state, and the agent can't touch anything outside the mounted workspace.

    • Declarative configuration

      A small .vb TOML file pins the harness, profile, features, extensions, and LLM provider. Commit-free, gitignored, reproducible.

    • Batteries included

      Profiles bundle Python, Go, Node, Playwright, a security audit toolkit, and more — resolved with dependency-aware feature layering.

    • Curated extensions

      A per-harness extensions catalogue of MCP servers, skills, agents, and bundles like ECC — installed at build time.

    • Host integrations

      Serena and claude-mem integrations wire host-side services into the container with automatic transport fallback — no manual Docker networking.

    • Credentials, not secrets

      Your OAuth tokens, API keys, GPG agent, and AWS credentials are forwarded into the container — never baked into the image.

    How it works in 30 seconds¶

    flowchart LR
    A[vibrate] --> B{.vb exists?}
    B -- no --> C[Wizard]
    C --> D[Save .vb]
    B -- yes --> E[Resolve spec]
    D --> E
    E --> F{Container?}
    F -- running --> G[docker exec]
    F -- stopped --> H[docker start + exec]
    F -- none, image exists --> I[docker run]
    F -- none, no image --> J[docker build] --> I
    G --> K([Agent])
    H --> K
    I --> K
    1. Resolve — vibrate reads the workspace .vb, overlays any CLI flags, and runs the wizard for anything still unset.
    2. Build — it generates a deterministic Dockerfile for the resolved spec and builds the image (only when one doesn't already exist).
    3. Launch — it runs or re-enters the container, mounting your workspace and forwarding credentials, then execs the harness's own CLI.

    See What happens on build and What happens on start for the full step-by-step.

    Status

    Vibrator is a Go program. Docker integration is done by shelling out to the docker CLI (no SDK dependency), so any Docker-compatible runtime works — Docker Desktop, OrbStack, Colima, Rancher Desktop, Podman, or native. Released under the MIT license.
